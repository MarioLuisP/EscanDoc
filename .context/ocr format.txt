● ¡Excelente pregunta! Sí, ML Kit Text Recognition retorna estructura jerárquica, no texto plano.

  📊 Estructura que retorna ML Kit:
  RecognizedText
    └─ TextBlock (bloques - párrafos/secciones)
        └─ Line (líneas dentro del bloque)
            └─ Element (palabras individuales)

  Cada nivel tiene:
  - text (el texto)
  - boundingBox (coordenadas x,y,width,height)
  - cornerPoints (4 esquinas exactas)
  - confidence (confianza)
  - recognizedLanguage

  ⚠️ Problema con 2 columnas/folletos:

  ML Kit NO entiende layout visual:
  - Lee de izquierda a derecha
  - NO respeta columnas
  - Mezcla texto de columna 1 y columna 2

  Ejemplo:

  Imagen original:
  ┌──────────┬──────────┐
  │ Columna 1│ Columna 2│
  │ Línea A  │ Línea X  │
  │ Línea B  │ Línea Y  │
  │ Línea C  │ Línea Z  │
  └──────────┴──────────┘

  ML Kit lee:
  "Línea A Línea X Línea B Línea Y Línea C Línea Z"

  Esperado:
  "Línea A Línea B Línea C Línea X Línea Y Línea Z"

  ¿Qué hace actualmente?
  Método usado: Propiedad .text de ML Kit
  - Toma el texto pre-concatenado que ML Kit retorna
  - NO accede a blocks/lines/elements manualmente
  - NO hace ordenamiento personalizado
  - NO detecta columnas

  ¿Cómo concatena ML Kit internamente?

  Comportamiento esperado de .text:
  - Une todos los TextBlocks con saltos de línea
  - Lee de izquierda a derecha, arriba a abajo
  - NO entiende layout visual (columnas, tablas)

  Implicaciones:

  ✅ Funciona bien para:
  - Documentos simples (1 columna)
  - Recibos verticales
  - Cartas, contratos lineales

  ❌ Problema esperado con:
  - Folletos de 2+ columnas (mezcla texto)
  - Diarios/revistas
  - Facturas con campos en posiciones específicas
  - Formularios con layout complejo
